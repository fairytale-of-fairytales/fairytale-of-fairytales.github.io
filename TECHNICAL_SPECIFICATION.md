# ТЕХНИЧЕСКОЕ ЗАДАНИЕ: «FAIRYTALE-OF-FAIRYTALES» — ЯДРО ДЛЯ ЖИВОЙ СКАЗКИ СКАЗОК

## 1. КОНЦЕПЦИЯ И СУТЬ
**Название:** Fairytale-of-Fairytales (Живая Сказка Сказок)
**Домен:** `https://fairytale-of-fairytales.github.io`

Это **генеративное нарративное пространство** — не сайт, а протокол для со-творения мифа.
*   **Ядро:** Динамическое «Сердцевинное Утверждение» (Core Statement), которое эволюционирует.
*   **Процесс:** Каждый посетитель может оставить «голос» — текстовый след, который становится частью растущей сказки.
*   **Структура:** Заложена графовая (нелинейная) модель. Каждый новый голос может быть ответом или ветвью любого предыдущего, создавая сеть смыслов.

## 2. ЦЕЛИ (ЗАЧЕМ ЭТО СУЩЕСТВУЕТ)
*   **Материализовать концепцию «виртуального государства»** как живой, дышащий цифровой организм.
*   **Создать простой и мощный ритуал** для «внутренних садиков» — совместное письмо как акт строительства общего мира.
*   **Получить минимально работающий, но бесконечно расширяемый артефозит**, который служит доказательством концепции и точкой сборки.

## 3. ТЕХНИЧЕСКИЕ ТРЕБОВАНИЯ

### 3.1. Стек и архитектура
*   **Фреймворк:** Quasar Framework (Vue.js 3) в режиме SPA (Single Page Application).
*   **Стиль (UI/UX):** Ultra-Minimalist / Digital Brutalism.
    *   Цветовая схема: преимущественно чёрный, белый, оттенки серого. Один акцентный цвет (например, `#7e57c2` для фиолетового — цвет мифа и трансформации).
    *   Акцент на типографике: моноширинный шрифт (например, `'Courier New', monospace`) для основного текста, sans-serif для интерфейса.
    *   Максимально пустое пространство (air). Никаких лишних линий, картинок, украшений.
*   **Хранение данных:** Git как база данных. Все голоса и связи хранятся в файле `public/data/story.json` в формате JSON. Каждое изменение — коммит.

### 3.2. Функционал Минимальной Жизнеспособной Версии (MVP v0.1)
**ГЛАВНЫЙ ЭКРАН:**
1.  **Core Statement (CS):** Крупный, динамический текст по центру экрана. Инициальное значение: `«Здесь начинается Живая Сказка Сказок. Твой голос — следующий абзац.»`
2.  **Лента Голосов (The Thread):** Под CS — вертикальная лента (последнее сверху) всех оставленных «голосов». Для каждого отображается:
    *   Текст (до 1000 символов).
    *   Условный идентификатор (например, короткий хэш).
    *   Дата в формате `DD.MM.YYYY HH:mm`.
    *   Кнопка `[→]` (ответить/ветвить).

**ИНТЕРФЕЙС ВЗАИМОДЕЙСТВИЯ:**
1.  **Кнопка «+» (Добавить Голос):** В правом нижнем углу экрана (FAB). По клику — появляется модальное окно/панель.
2.  **Панель добавления:**
    *   Текстовое поле (`<textarea>`) с плейсхолдером: `«Впиши свой след в историю...»`
    *   Кнопка «Оставить След». При нажатии:
        *   Текст валидируется (не пустой, не превышает лимит).
        *   В фоновом режиме (через GitHub API или серверную функцию) создаётся новый коммит в `story.json`, добавляющий объект голоса.
        *   Лента мгновенно обновляется, новый голос появляется сверху.
        *   Панель закрывается.

**ЗАЛОЖЕННАЯ АРХИТЕКТУРА ДЛЯ БУДУЩЕГО (v0.2+):**
*   В данных каждый голос имеет уникальный `id` и опциональный `parent_id` (для создания ветвей).
*   Визуализация графа связей (в будущем — через `vis-network` или `cytoscape`).

## 5. ФИЛОСОФСКИЕ ПРИНЦИПЫ (ЗАКОНЫ СКАЗКИ)
1.  **Простота — высшая сложность.** Интерфейс должен исчезать, оставляя только мысль.
2.  **Каждый коммит — это акт мифотворчества.** История Git — это и есть сюжет.
3.  **Код — это тоже часть сказки.** В комментариях приветствуются лирические отступления, объясняющие «почему».
4.  **Система должна расти органично,** как живой организм, а не строиться по чертежу.
